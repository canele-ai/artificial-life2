# Problem Specification

> Generated by Phase 1.5 Problem Refinement panel (3 proposers + reviewer).
> Revised in Phase 1.6/2.0 to adopt a **VLM-as-judge** final-scoring metric.
> Supersedes aspirational Phase 1 decisions. Ground truth for Phase 2.

## Refined Research Question

Discover simulation parameters (in Lenia and Flow-Lenia) whose rollouts score
highest under a **multimodal-LLM rubric** covering the 5 operational tiers
of life-like behavior (Existence, Agency, Robustness, Reproduction,
Coherence), judged by a frozen Claude Sonnet with temperature=0 on 16
held-out seeds, measured against a pinned ASAL Sep-CMA-ES zero-anchor
baseline.

Single goal: **beat the pinned ASAL baseline on the held-out split by METRIC
> 0 with seed-paired Wilcoxon p < 0.05.**

## Metric

- **Name:** `lcf_judge_heldout` (Life-Cycle Fidelity, VLM-judge, held-out)

- **Inner-loop search signal (orbit's freedom):** orbits may compute any
  CLIP-based scalar during `search()` — ASAL supervised target, ASAL
  open-endedness, ASAL illumination, custom CLIP heads, whatever. Orbits
  may NOT call the Claude judge API during `search()` (process-isolated;
  any network call to `api.anthropic.com` from inside the search container
  is blocked).

- **Final-scoring formula (the campaign metric):**
  ```
  For each held-out seed s ∈ S_test (|S_test| = 16):
    rollout  = substrate(θ, s)                              # [T=256, H=128, W=128, C]
    picks    = rollout[[0, 63, 127, 191, 255]]              # 5 ordered frames
    strip    = concat_horizontally(picks)                    # [128, 640, 3] single image
    scores_k = judge(strip, rubric)  for k ∈ {1, 2, 3}       # 3 independent judge calls
    scores   = median across k of each tier                 # robust to judge variance
    tier_scalar(s) = geometric_mean(scores)                 # fails any tier → near-zero

  LCF_judge(θ) = trimmed_mean_10pct(tier_scalar across S_test)
  METRIC(θ)    = LCF_judge(θ) − LCF_judge(pinned ASAL baseline)
  ```

  Judge: **Claude Sonnet 4.6** (`claude-sonnet-4-6`), temperature=0,
  `max_tokens=256`, system-prompt + rubric SHA-pinned in
  `research/eval/judge/rubric.md`.

  Three judge calls per (θ, s) for variance reduction (median); total
  judge calls per orbit = 16 seeds × 3 = 48 per θ × 1 θ (best_params only)
  = **48 judge calls per orbit**.

- **Direction:** maximize. Expected range ≈ [−0.1, +0.3]. `> 0` means beats
  ASAL baseline by a single-judge-consistent margin.

- **Provenance:** the VLM-as-judge paradigm is standard in frontier
  benchmarks (LMSys Arena, HELM, MLCommons). This campaign applies it to
  artificial life — novel combination. Inner-loop CLIP scoring inherits
  directly from ASAL (Kumar et al. 2024, arXiv:2412.17799).

- **Target:** no hard target. Pass bar = METRIC > 0 at seed-paired Wilcoxon
  p < 0.05 over the 16 held-out seeds. Stretch: METRIC > 0.05.

- **Known pitfalls (and guards):**
  - *Judge variance:* mitigated by median-of-3 judge runs + temperature=0.
  - *Judge prompt-injection via images:* mitigated by a structured-output
    rubric that requires numeric-only tier scores, and by rejecting judge
    responses that don't parse as JSON.
  - *Inner-loop-signal ≠ judge-signal divergence:* this IS the research
    question. Orbits that find search signals well-correlated with the
    judge rubric will win.
  - *Chaotic-high-CLIP-novelty rollouts:* the Coherence tier in the rubric
    explicitly penalizes noise/chaos.
  - *Dead/exploded patterns:* the Existence tier penalizes these directly.
  - *Seed lottery:* 16 held-out seeds + Wilcoxon test.
  - *Judge as oracle:* process-isolated; orbits cannot import the judge
    client or call api.anthropic.com.
  - *Judge-model drift (frontier model updates):* pin exact model version
    string; re-run ASAL baseline whenever the judge model changes (new
    `eval-vN` tag).

## The Rubric (the research contract)

Frozen verbatim in `research/eval/judge/rubric.md`. Draft:

```
You will see a horizontal strip of 5 frames from an artificial-life
simulation, left-to-right in temporal order (time steps 0, 63, 127, 191, 255).

Rate the trajectory on each of 5 dimensions. Each score must be a float in
[0.0, 1.0]. Return ONLY a JSON object, no prose:

{
  "existence":    <float>,   // Is there a coherent, spatially localized pattern
                             // distinct from the background? (not blank, not filling
                             // the whole field, not uniform noise)
  "agency":       <float>,   // Does the pattern MOVE or CHANGE SHAPE in a structured
                             // way across frames? (translation, rotation, pulsation,
                             // deformation — not random flickering)
  "robustness":   <float>,   // Does the pattern PERSIST and maintain a recognizable
                             // identity across all 5 frames? (not dissolving, not
                             // exploding, not replaced by something unrelated)
  "reproduction": <float>,   // Do additional similar patterns appear, or does the
                             // pattern visibly divide/bud? (0 if no offspring, 1 if
                             // clear offspring visible in later frames)
  "coherence":    <float>    // Is the pattern visually structured and NON-CHAOTIC?
                             // (0 = noise/mush, 1 = crisp geometric structure)
}

Be strict. Uniform blobs, dissolved patterns, chaotic noise, and empty
fields should all score below 0.2 on most dimensions.
```

## Solution Interface

```python
from typing import Literal
import jax

def search(
    substrate_name: Literal["lenia", "flow_lenia"],
    seed_pool_train: jax.Array,            # shape [16], disjoint from S_test
    budget: dict,                          # {"wall_clock_s": 1800}  hard 30-min cap
    rng: jax.Array,                        # PRNGKey
) -> dict:
    """
    Returns
    -------
    {
      "best_params":  jax.Array,    # [D] float32
      "archive":      jax.Array,    # [<=16, D] float32  (diagnostic only)
      "search_trace": dict,         # {"best_proxy_per_gen": [G], ...}
    }

    Constraints
    -----------
    - Bit-identical determinism under fixed rng.
    - Must respect budget["wall_clock_s"]; evaluator SIGKILLs at cap.
    - May compute any CLIP-based scalar as search-time fitness proxy.
    - May NOT call api.anthropic.com, api.openai.com, or any judge API.
    - May NOT read files under `research/eval/judge/` or `research/eval/heldout/`.
    - CLIP weights available from `/cache/hf` (pre-populated Modal Volume).
    """
```

`prompt_bank_train` / `search_fm` parameters from the earlier design are
REMOVED — no text prompts, no multi-FM ensemble at scoring time. Orbits
have full freedom over search-time proxy choice.

## Evaluation Procedure

1. Evaluator receives `solution.py` path + run_id. Instantiates two Modal
   containers:
   - **search container**: A100, CLIP weights cached, network egress blocked
     to api.anthropic.com / api.openai.com.
   - **judge container**: CPU, Anthropic SDK, `anthropic-api-key` secret
     mounted, `claude-sonnet-4-6` pinned.
2. search container calls `solution.search(...)` with `seed_pool_train` and
   `budget = {"wall_clock_s": 1800}`. SIGKILL at cap. On timeout, last
   checkpointed `best_params` is used; if none, METRIC = 0.0.
3. search container rolls out `substrate(best_params, s)` for each
   `s ∈ S_test` (16 rollouts). Generates horizontal 5-frame strips.
4. search container saves strips to `/tmp/strips/{run_id}_{s}.png`.
5. judge container loads each strip, calls `judge(strip, rubric) × 3`,
   records scores as JSON.
6. Evaluator aggregates per-tier median, computes `tier_scalar(s)` as
   geometric mean, computes `LCF_judge(θ)` as 10%-trimmed mean.
7. Evaluator looks up `LCF_judge(ASAL_baseline)` from frozen
   `research/eval/baseline/baseline_score.json`.
8. Prints `METRIC=<value:.6f>` and `METRIC_COMPONENTS={...}` (per-tier
   medians, per-seed scalars, Wilcoxon p-value, judge version).

## Baselines

| Name | Expected LCF_judge (raw, pre-subtraction) | Source |
|------|-------------------------------------------|--------|
| Trivial random params (single sample) | ~0.05 ± 0.05 | Lenia/Flow-Lenia random init |
| Random search (pop 64 × gens 200, best by inner CLIP-OE) | ~0.15 | 30-min budget-matched baseline |
| **ASAL Sep-CMA-ES + supervised-target-CLIP (pinned)** | **~0.25–0.40** (to measure at init) | ASAL repo @ pinned SHA, CLIP ViT-B/32, Sep-CMA-ES from evosax, pop=16, σ=0.1, gens=200, PRNGKey(0) |
| SOTA aspirational | +0.05 over ASAL baseline (our target) | our campaign |

The ASAL baseline uses its own original metric during search
(supervised-target-CLIP), then is judged by Claude on the held-out seeds.
This is the un-gameable zero anchor.

## Feasibility Assessment

- **Estimated eval time per orbit (A100 search + CPU judge):**
  - `search()`: 30 min A100 (hard cap, user-enforced)
  - Held-out rollouts on A100: ~10 s
  - Judge calls: 16 seeds × 3 runs = 48 Claude Sonnet calls @ ~4 s each,
    parallelized to 8 concurrent → **~25 s wall-clock**
  - Container overhead + logging: ~60 s
  - **Total per orbit: ~32 min**
- **Estimated eval cost per orbit:**
  - A100: ~0.55 A100-hr × $1.50 = ~$0.83
  - Judge: 48 Sonnet calls × ~1500 input tokens + 100 output tokens
    ≈ 77K input + 5K output tokens × ($3/M input + $15/M output) ≈ **$0.31**
  - **Total per orbit: ~$1.14**
  - Full 50-orbit campaign ≈ $55–70.

## Disk Hygiene (enforced at evaluator level)

- **Rollout tensors:** never saved raw. Only leaderboard top-10 orbits'
  per-seed rollouts are encoded as **mp4** (h.264 CRF 28, ~500 KB each).
  Non-leaderboard orbits keep only 1 teaser mp4.
- **Frame strips (judge inputs):** PNG at 128×640, deleted after judge call
  unless orbit is in top-10.
- **GIFs for the Campaign Issue / viz:** palette-reduced, ≤ 64 frames,
  ≤ 320×320, target ≤ 300 KB. 1 teaser per orbit; more only for top-10.
- **Per-generation trace:** scalars only — `best_proxy_per_gen`,
  `mean_proxy_per_gen`, timestamps. NO θ histories.
- **Judge response logs:** JSONL, gzipped, ~2 KB per orbit.
- **FM weight cache:** Modal Volume `hf-cache` mounted read-only at
  `/cache/hf`. Weights downloaded once at campaign bootstrap.
- **JAX JIT cache:** Modal Volume `jax-jit-cache` at `/cache/jax`.
  `JAX_COMPILATION_CACHE_DIR=/cache/jax`.
- **Orbit worktree lifecycle:** `git worktree remove --force` on merge or
  `label:dead-end`. Retain only `label:winner`.
- **Artifact budget per orbit: ~10 MB** (mp4s + 1 GIF + scalars + JSONL).

## Novelty Operationalization

The campaign's research novelty is: **VLM-as-judge for artificial-life
search**. Every orbit is an independent hypothesis about how to efficiently
search ALife space when the judge is a multimodal LLM rubric. Permutation-
gap and prompt-bank design elements from earlier drafts are dropped —
they're no longer needed now that the rubric is the research contract.
